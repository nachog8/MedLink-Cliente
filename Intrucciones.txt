Instrucciones para el Desarrollo del Front-End

1. Estructura de Componentes en Pages

    Si van a crear una nueva página en la carpeta pages, por favor:
        - Al crear los componentes específicos de esa página, guárdenlos en la carpeta components.
        - No dejen estos componentes sueltos en la carpeta components; en su lugar, creen una carpeta con el nombre de la página y guarden dentro todos los componentes que se usen solo en esa página.
            -   Ejemplo: Si están trabajando en la página Profile, dentro de components creen una carpeta llamada profile y guarden allí todos los componentes relacionados, como card-profile.tsx, profile-header.tsx, etc.
2. Convenciones de Nombres de Componentes

    Para el nombre de los archivos de componentes:
        - Sigamos la convención de todo en minúsculas y, si es un nombre compuesto, usemos guiones (-) para separar palabras.
            Ejemplos: card-profile.tsx, user-avatar.tsx, settings-form.tsx.
        - Dentro del archivo pueden exportar la función con el nombre en PascalCase si lo prefieren, por ejemplo:

                                export default function CardProfile() {
                                // contenido del componente
                                }
3. Naming en las Rutas
    Para las páginas en las rutas (pages), no es necesario asignarles un nombre específico.
    Dejemos el nombre genérico como "Pages" en el código de rutas y componentes, de modo que se facilite su mantenimiento y consistencia en toda la aplicación.
4. Creación de Ramas desde Development
    Antes de crear una nueva rama de trabajo, asegúrense de actualizar su rama development:

        git checkout development
        git pull origin development

    Luego, creen la nueva rama desde development usando el nombre de la funcionalidad en la que van a trabajar.

        Ejemplo:
        git checkout -b feature/profile-page
        
Asegúrense de que el nombre sea descriptivo de la funcionalidad o cambio que implementarán.
Siguiendo estas instrucciones, mantendremos una estructura ordenada y fácil de seguir para todos. ¡Gracias por su colaboración!
